## Group reports / standups

Listening to other groups perform their standups put into perspective the speed our team is able to work at. <!--more-->Others were getting their development environments set up while we were already busy assigning work and creating issues. Relatively speaking, they were experiencing the same things we had already gone through.

## Issue issues

However, our speed was not fully to our advantage. We had overlooked rebuilding the source code of the repository and created an issue for a feature that was already implemented. Once we realized that, the issue was closed. It is difficult to be a user and be fully up to date on the status of what still needs to change because the nightly build is far newer than the versions that the maintainers release on Docker.
